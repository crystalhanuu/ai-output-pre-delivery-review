# Example 01 — 调研报告审核(真实样本)

> 📌 **状态**:基于真实样本的使用示例。
> 本示例展示 Skill 在审核一份"德国 dating APP 商业画布调研"时的完整输出。
> 输入约 5000 字,内容包含公司背景、商业画布、用户画像、营收数据、推广渠道等。

---

## 用户输入

用户从 Lark 文档直接复制粘贴了一份德国 dating APP "LOVOO" 的商业画布调研报告(约 5000 字,包含公司沿革、商业画布九模块、用户画像、营收数据、推广渠道、会员费表等),没有添加触发指令,**直接回车**。

Skill 自动识别为应审核的 AI 输出,主动启动了审核流程。

---

## Skill 输出

### 审核范围说明

- 输入类型:调研报告(给老板/团队看的德/法 dating app 商业画布对比研究)
- 原始任务上下文:已提供 — 任务核心是"德法 dating app 对比,侧重 APP 站内和商业化;德国深做商业画布,法国侧重品牌差异化;法国如果是匹配类型不用太深"
- 审核基于原文 Markdown,未做清洗
- 本审核不做事实核查,公司历史/营收/收购等数字一律标"待验证"

⚠️ 一个前置事实:法国前三名(Meetic / happn / adopte)只列了 APP 名字,完全没有内容。本审核默认你是想审"目前已写的部分能不能用",但请确认是否还有未粘贴的法国部分。

---

### 格式完整性检查

| 检查项 | 状态 | 发现的问题 | 人类判断点 |
|---|---|---|---|
| 结构 | 不足 | 单 APP 内章节齐(KP/KA/VP/CR/CS/KR/CH/Cost/Revenue),但法国整段缺失;缺执行摘要、缺横向对比表、缺"侧重点区别"专节 | 法国是真的没做,还是没粘贴? |
| 结论 | 缺失 | 三个 APP 各自一节,但没有任何综合结论。任务要的"侧重点区别"完全没体现 | 这份报告是要给老板看决策,还是只做"原始材料整理"? |
| 依据 | 不足 | 数据量很丰富,但多处"预估?人""品宣成本:?""Chatter 或 Robot?""兑换比例未知"直接出现在交付物里。VP 部分(如"贩卖近距离且易得的陪伴")是作者推断,但没标 | 这些悬而未决项是补齐再交,还是标"待确认"交出去 |
| 来源 URL | 不足 | 只有 appannie / Trustpilot / YouTube 等少数标了出处;营收/留存/用户画像/广告数据大量无来源 | 是否每个数字必须挂一个来源 |
| 行动项 | 缺失 | 整份文档没有"对我们产品的启示"或"建议借鉴/避免" | 老板拿到这份是要做决策还是只看资讯 |
| 验证项 | 缺失 | 文中散落多个"?"和"未知",但没汇总成"需要复核的清单" | 是否在文末加一节"待验证 / 仍需确认" |

---

### 蓝军风险清单

按交付影响力排序。

**1. [影响交付] 任务核心是"德法对比",但法国部分完全空白**

任务原话:"侧重点区别,APP 站内 商业化部分","德国前三名商业画布... 法国前三名商业画布(常规品牌 宣传和品牌差异化)"。

现状:法国部分只有"Meetic / happn / adopte"三个名字 + 一张图占位,0 内容。文档标题还叫"德国_商业画布"(德国和法国应该并列)。

直接结果:交付物名义上是德法对比,实质只交了一半,且这一半的"对比对象"不存在,所以"侧重点区别"这个核心任务根本没法呈现。

[建议方向] 要么补法国 3 个 APP 的轻量画布(品牌 + 商业化 + 1 张差异化对比表);要么和老板对齐"先交德国,法国 v2 补"并把文档标题/范围改清楚

**2. [影响交付] 没有横向对比表,也没有"侧重点区别"专节 — 任务真正要的东西没产出**

LOVOO 月营收 67 万 / Snoggle 15.8 万 / Viluu 13.5 万;LOVOO ARPPU $23 / Snoggle $190 / Viluu $145;LOVOO 大众轻氪 / Snoggle、Viluu 极小众重氪 — 这些差异本身就是"侧重点"。但报告把它们留在各自小节里,从来没有把三者放在一张表里看。

老板拿到这份报告,会反问的第一句话大概率是:"所以德国前三的差异是什么?",而报告没回答。

[建议方向] 加一个 3-5 行的横向对比表(月营收 / ARPPU / 用户量 / 商业模式画像 / Fake 驱动倾向);再加一段 5-10 行的"侧重点结论"

**3. [影响接受度] 三个 APP 都暴露了 Fake/Chatter/骗局信号,但这一最具洞察的发现没被抽出来**

- LOVOO:Fake 用户问题
- Snoggle:"骗局和机器人""聊天机器人内幕"(还放了 YouTube 链接)、Trustpilot 3.3 分
- Viluu:"假账号、代聊配假照片"、"专业聊天机构 Peerwise.de"、Trustpilot 差评

结合 ARPPU 数据:Snoggle($190)、Viluu($145)显著高于 LOVOO($23) — 这几乎可以推断:Snoggle/Viluu 的核心商业模式是 chatter 驱动的小众重氪,LOVOO 是品牌驱动的大众轻氪。这是这份报告里最有商业洞察的发现,但被散落在各自的"评论反馈"里,从未作为结论提出。

给老板看时如果不点这层,等于浪费了已经发现的素材。

[建议方向] 单独写一节"商业模式判断"或"伦理/合规风险",把这层指出来。注意措辞用"疑似 / 用户评价显示 / 第三方报道",不要直接下定论

**4. [影响可信度] 数字单位多处写错或重复,容易在交付现场被抓到**

- "近365天营收 1,219 万万美金" — "万万"显然是错的(应为"1,219 万美金"或"1.219 亿美金")
- "近365天营收 $155 万美金" — "$" 和"美金"重复
- Viluu 金币列底部出现孤立的 "73" — 像是被截断的内容
- "白营官方号" — 应是"自营官方号"(LOVOO、Snoggle 部分都有)
- "Rette meehr" — 疑似 "Rette mehr"
- LOVOO 旗下 APP 表里"meet me"应该是"MeetMe"(品牌大小写)

数字错误在老板眼里直接 = 整份报告不可信。

[建议方向] 交付前过一遍 — 重点过营收数字和金额单位

**5. [影响可信度] 大量数据无来源 / 不少关键值是"?"**

- 营收、留存、ARPU、CPI、广告平台份额这些数字 — 没标采集工具和时间(只有"近 365 天",起止日期不明)
- 多处直接写"预估 10 人""预估 20 人""品宣成本:?""Chatter 或 Robot ?"
- 估算的"管理成本 10%""5 万美金/员工/年" — 假设来源不明

老板会问"这些数字哪来的""为什么估 10 人""品宣成本怎么会是问号"。

[建议方向] ① 在文档开头加"数据口径"小节(数据源 + 时间窗口 + 估算假设);② 把所有"?"统一收到末尾"待确认清单",并指明由谁补齐

**6. [影响清晰度] 三个 APP 的公司背景深度严重不平衡,横向不可比**

- LOVOO:深度极高(MyYearbook 历史链、Parship Meet 营收、旗下 APP 矩阵、vPaaS 业务,几千字)
- Snoggle:中等(注册地变更、董事变更)
- Viluu:极简(注册资金 300 EUR、董事换人)

老板看会觉得"为什么 LOVOO 写这么细,另外两个只有几行?是研究深度不够还是另外两个真的没料?"

[建议方向] 要么把 LOVOO 浓缩到与另两家可比的篇幅,把"vPaaS"等溢出信息放附录;要么补另外两家的公司背景到同等深度;别让深度差成为可信度的减分项

**7. [影响清晰度] "消费路径(CR)"在三个 APP 间格式不统一,横向对比困难**

- LOVOO 写"基本功能 → 会员支付"
- Snoggle 列每个动作消耗几金币
- Viluu 也列金币但格式又不同

应该是同一画布维度,但你给了三种不同写法,导致横向看"哪些 APP 有哪些路径"很费劲。

[建议方向] 做一张统一的 CR 矩阵表:列=APP,行=主要付费触点(注册解锁 / 划卡限额 / 破冰 / 谁喜欢我 / 私照 / 礼物 / 消息 / Boost / SuperLike),格子里写"金币/会员/免费/无此功能"

**8. [影响接受度] 任务边界"法国如果是匹配的话就不用搞太深"在文档里没归档**

这是决定法国部分深度的关键约束,但报告里没有任何"基于 X 边界,法国采取 Y 深度"的声明。老板看时会问"为什么法国这么浅",你需要随时能搬出当时的边界。

[建议方向] 文档开头加一段"研究范围与深度边界",把这条原始约束写进去

**9. [影响交付] 大量 "[图片]" 占位 — 实际交付时图片是否会渲染**

报告里多处 [图片](广告平台份额、用户画像、广告素材截图、热力图等),这些在 Lark 里可能正常,但导出 PDF / 截屏给老板时是否会丢?如果丢图,关键数据(平台份额、广告素材)等于没交。

[建议方向] 交付前确认导出格式;关键数据图(平台份额、ARPU/ARPPU 对比)考虑改用表格+数字承载,即使丢图也不影响阅读

**10. [影响可信度] 时间口径缺失**

- 数据采集时间窗口(只说"近 365 天",起止日不明)
- 公司信息时点(LOVOO 写 "2024 年 Parship Meet 营收 3.75 亿欧元",这是最新数据吗?)
- 调研完成时间

老板看时不知道"这是过去什么时候的快照"。

[建议方向] 文档开头加"数据采集日期 / 数据窗口 / 调研完成日"

**11. [影响可信度] 几个高频被追问的数字需先核(标"待验证")**

- ParshipMeet 2024 营收 3.75 亿欧元、利润率 16%
- "vPaaS 为全球排名前 50 的社交应用中的 7 款提供视频"
- LOVOO 旗下 APP 收购年份(2016 Skout、2017 if(we)、2017.9 Lovoo、2018 eharmony)
- "巴拿马音乐节"是不是 Parookaville?这是德国一个真实的电音节,叫"Parookaville";"巴拿马音乐节"听起来像翻译错误
- Snoggle 历年公司迁址链(温特图尔 → 黑里绍 → 黑吉斯维尔)

这些是老板/同事最容易现场拿出手机核对的数字。

[建议方向] 交付前过一遍数字 + 列出来源链接(Crunchbase / 公司官网 / Statista)

**12. [影响清晰度] LOVOO ARPPU $23 vs Snoggle $190 vs Viluu $145 这个核心商业差异未被解读**

ARPPU 差 6-8 倍是非常显著的信号。可能含义:

- LOVOO 走会员订阅 + 大基数,人均价值低但用户多
- Snoggle / Viluu 走金币消费(消息/礼物按次),小基数重氪,可能有 chatter 推动

但报告把这三个数字分别埋在各自的 Revenue 节里,没拿出来做对比解读。这是商业画布报告本该承担的工作。

[建议方向] 在"侧重点区别"专节里,把 ARPPU / 商业模式画风 / Fake 倾向连起来讲

---

### 人类判断点

跨条目需要先拍板的:

1. 法国部分还做不做?做到什么深度? — 这决定整份能否按"对比"交付
2. 是否补"侧重点区别"专节 — 任务的真正交付物
3. Fake/Chatter 的判断是否要在报告里点明 — 涉及措辞分寸(直接说 vs 说"用户评价显示")
4. 悬而未决项("?"、未知)的处理 — 补齐再交 / 标待确认交 / 直接删
5. 数据来源标注的力度 — 每条都标 vs 只标关键数字
6. 图片渲染 — 交付用什么格式,会不会丢图
7. 交付时间 — 周五前的硬约束是否仍成立

---

### 可选修改方向

整体性建议:

- 这份报告原料很丰富,但加工度不够。已经收集到了三个 APP 的关键数据,差的是"对比 + 结论 + 行动启示"。如果给老板看,建议在最前面加一页执行摘要 + 一张横向对比表,光这两件事就能让交付质量上一个台阶,不需要重做底层调研
- 把所有"?"、"未知"、"预估"集中放到末尾的"待确认清单",并指出谁来补齐 — 这样老板知道哪些是已确认 vs 待补,避免把"?"当成结论
- ARPPU + Fake/Chatter 这层洞察是这份报告里最有商业价值的,建议升级成独立的一节"商业模式判断",而不是埋在评论反馈里
- 法国部分如果时间不够,至少交一张"德 vs 法 dating 市场结构差异"的对比表,把任务核心至少补齐到 MVP

---

### 仍需确认

- 法国部分是真的没写,还是没粘到对话里?
- 这份报告的最终交付对象是谁(老板 / 产品团队 / 自存档)?不同对象对结论/行动建议的需求差异极大
- 数据底座是 data.ai(原 appannie)+ Sensor Tower 吗?调研窗口是哪一段?
- 是否允许在报告中直接出现"疑似 fake / chatter 驱动"这类判断,还是必须更保守
- 交付截止是哪一天

## 这个示例展示了 Skill 的哪些能力

1. **零触发词自启动**:用户没说"帮我审核",Skill 自主识别要审核
2. **跨章节信号关联**:把分散在三个 APP 章节的 Fake/Chatter 信号 + ARPPU 数据串联成商业洞察
3. **守住"标可疑不判真伪"边界**:"巴拿马音乐节疑似 Parookaville 翻译错"用"疑似",不上网查
4. **影响维度标签**:未使用 [高][中][低],按"影响什么维度"分类
5. **[建议方向] 不替用户决定**:所有建议都是方向,不是结论
6. **抓字符级细节**:"1219 万万美金""白营官方号""Rette meehr"等错字
7. **预判老板会怎么追问**:多条风险都以"老板会问"作为锚点,贴近真实交付场景
